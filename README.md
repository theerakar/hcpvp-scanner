HCPVP Scanner **Hybrid Context-Aware Privacy Vulnerability Prevention** Final Year Project | BSc (Hons) Cybersecurity | APIIT Colombo / Staffordshire University UK ## Overview An automated DevSecOps web dashboard that detects GDPR and CCPA privacy violations in Java/Spring Boot source code using static AST analysis (JavaParser) and dynamic runtime monitoring. This tool implements Modules 1 and 3 of the HCPVP framework proposed in: Madhushanie et al. (2025). ## Privacy Rules Implemented | Rule ID | Description | Regulation | |---|---|---| | PRIV-SEC-01 | No password hashing | GDPR Art.32 + CCPA §1798.150 | | PRIV-SEC-02 | Hardcoded secrets in source | GDPR Art.32 + CCPA §1798.150 | | PRIV-DEL-01 | Missing DELETE endpoint | GDPR Art.17 + CCPA §1798.105 | | PRIV-DES-01 | Missing access control | GDPR Art.25 + CCPA §1798.150 | | PRIV-SEC-05 | SQL string concatenation | GDPR Art.32 + CCPA §1798.150 | | PRIV-MIN-01 | Excessive PII collection | GDPR Art.5 + CCPA §1798.100 | ## Regulatory Framework - GDPR (EUR-Lex, 2016/679) — Articles 5, 7, 17, 20, 25, 32 - CCPA (California DOJ, 1798.100–.150) - CNIL GDPR Developer Guide (Sheets 1, 4, 6, 7, 8, 13, 14) - OWASP Top 10 Privacy Risks (P1, P2, P6, P10) ## Tech Stack - Java 17, Spring Boot 3.3 - JavaParser 3.28.2 (static AST analysis) - Thymeleaf, Bootstrap 5 - Privacy Risk Fusion Engine: PRS = αS + βD (α=0.8, β=0.2) ## Status > Implementation in progress. Code will be committed following midpoint evaluation submission. ## Supervisor Project conducted under supervision at APIIT Colombo, affiliated with Staffordshire University, UK.
